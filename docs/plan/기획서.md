# **기획서**

# **\[Omni-Dash\] 금융 투자 대시보드 자동 생성 서비스**

# **0\. 개요**

**"Omni-Dash는 복잡한 금융 데이터를 업로드만 하면 검증된 분석 규칙을 바탕으로 초보 투자자에게 최적화된 맞춤형 투자 대시보드를 자동 생성해주는 서비스입니다."**

# **1\. 기획 배경 및 문제 정의**

초보 투자자(개미 투자자)는 정보의 홍수 속에서 자신의 투자 현황을 객관적으로 파악하는 데 큰 어려움을 겪고 있습니다. Omni-Dash는 이들이 겪는 세 가지 핵심 페인 포인트(Pain Point)를 해결합니다.

| 구분 | 초보 투자자의 현실 (Problem) | Omni-Dash의 해결책 (Solution) |
| :---- | :---- | :---- |
| **데이터 관리** | MTS, 엑셀, 노트 등에 기록이 파편화되어 있어 통합적인 계좌 상태 파악 불가 | One-Click 통합: 엑셀/CSV 업로드 즉시 하나의 대시보드로 통합 렌더링 |
| **분석 역량** | MDD(최대 낙폭), 포트폴리오 변동성 등 객관적 지표를 모르고 감이나 외부 정보에 의존 | 객관적 지표 자동화: 검증된 룰셋(Skills.md)이 정확한 공식으로 자동 계산 |
| **도구의 한계** | 전문 툴(BI 도구)은 다루기 어렵고, 기본 증권사 앱은 제공하는 시각화 정보가 제한적임 | AI 맞춤형 시각화: 데이터 성격에 맞춰 가장 직관적인 차트를 스스로 구성 |

**핵심 가치:** 재사용성, 검증 가능성, 협업성(비개발자 수정 가능), 모듈성.

# **2\. 핵심 설계 원칙 및 차별화 전략**

Omni-Dash의 가장 큰 차별점은 엄격한 규칙 문서(Skills.md)를 코어 엔진으로 삼는 LLM입니다.

**분석 규칙의 통제 (What):** AI가 마음대로 수치를 창작하지 못하도록 모든 지표 계산식과 분석 기준을 Skills.md에 명시합니다.

**구현의 자동화 (How):** LLM은 오직 데이터를 화면에 어떻게 예쁘게 보여줄지(바이브 코딩)에만 집중하여 프론트엔드를 실시간으로 생성합니다.

# **2.1 설계 원칙**

| 원칙 | 의미 | 구체적 적용 |
| :---- | :---- | :---- |
| Decoupling | 규칙(What)과 구현(How) 분리 | Skills.md는 분석/시각화만 정의, 라이브러리/프레임워크는 구현 힌트 수준 |
| Single Source of Truth | 지표는 한 곳에서만 정의 | 시맨틱 레이어 외 지표 계산/표시 금지 |
| Progressive Disclosure | 핵심은 본문, 세부는 분리 파일 | 자주 쓰는 규칙만 본문, 자산군별 임계값/예시는 references/로 분리 |
| Guardrails First | 안전한 기본값과 금지 행위 우선 | 금지 행위, 면책, 통계적 유의성 경고 강제 |

# **2.2 핵심 의사결정 및 차별화 포인트**

| 의사결정 | 본 기획의 선택 | 이유/차별화 포인트 |
| :---- | :---- | :---- |
| 출력 강도 | Quick/Standard/Full 3단 모드 분기 | "MDD만 알려줘" 같은 단순 질의에도 풀 파이프라인을 강제하는 사용자 경험 파괴 방지 |
| 임계값 지표 정의 | 자산군별 가이드 표 시맨틱 레이어 단일화 | 채권 0.5와 암호자산 1.0은 의미가 완전히 다름. 단일 임계값이 아닌 상대적 해석 프레임워크 제공 |
| 인사이트 신뢰성 | \\\[ref: metric=, period=, n=\\\] 추적성 강제 | LLM 환각 차단 및 검증 가능성 확보 |

# **3\. 상세 핵심 기능**

초보 투자자가 복잡한 설정 없이도 전문가 수준의 인사이트를 얻을 수 있도록 4가지 핵심 기능을 제공합니다.

| 핵심 기능 | 세부 작동 방식 | 초보 투자자 체감 효용 |
| :---- | :---- | :---- |
| **1\. 지능형 데이터 정규화**(Smart Normalization) | 형태가 제각각인 업로드 데이터를 시스템이 인식하여, 표준 스키마로 강제 변환 및 결측치 자동 보정 | "복잡한 엑셀 서식 맞추기에 시간 낭비할 필요 없이, 파일만 드래그 앤 드롭하면 끝납니다." |
| **2\. 표준 지표 계산 엔진** (Metric Engine) | 수익률, 변동성 등 필수 지표를 외부 API나 AI의 임의 판단이 아닌, 엔진 내 확정된 공식으로만 산출 | "계산 실수나 AI의 환각(거짓 정보) 없이, 내 계좌의 진짜 상태를 정확히 볼 수 있습니다." |
| **3\. 맞춤형 차트 자동 매핑**(Auto-Visualizer) | 시계열 데이터는 꺾은선, 자산 비중은 파이/트리맵 등 데이터 속성에 맞춰 최적의 UI 컴포넌트 자동 매핑 | "차트 설정을 고민할 필요 없이, 가장 이해하기 쉬운 형태의 그래프가 알아서 뜹니다." |
| **4\. 객관적 인사이트 요약** (Guardrailed Insights) | 주관적 판단(매수/매도 추천)을 철저히 배제하고, 차트에 나타난 팩트 기반의 현황 요약 리포트 제공 | "유튜브나 커뮤니티의 맹신에서 벗어나, 내 데이터가 말해주는 객관적인 시장 흐름을 읽게 됩니다." |

# **3.2 해커톤 목표 산출물**

해커톤 기간 내에 달성해야 할 목표 산출물과 우선순위는 다음과 같습니다.

| 산출물 | 역할 | 우선순위 |
| :---- | :---- | :---- |
| Skills.md | 핵심 규칙 명세 | PO |
| 참고 데이터 셋 | 5개 대표 입력 형태 데이터 | PO |

| CHANGELOG.md | 규칙 변경 이력 | P1 |
| example outputs | Skills.md 적용 결과 샘플 (PDF/스크린샷) | P1 |

# **4\. 해커톤 구현 전략**

주어진 시간 내에 혁신적인 결과를 내기 위해, AI 기반 바이브 코딩(Vibe Coding)을 극대화합니다.

* **Core Engine (Skills.md):** 지표 산출식, 시각화 가이드라인, 프롬프트 제한 사항 등을 담은 약 400라인 내외의 마크다운 룰셋 작성.  
* **Real-time Rendering:** 사용자가 데이터를 업로드하면 시스템이 Skills.md를 참조하여 React/Vue 기반의 대시보드 화면 코드를 즉석에서 짜고 렌더링.  
* **3-Step 리포트 모드:** 초보자의 눈높이에 맞춰 ① 한눈에 보는 핵심 요약 ➔ ② 종목별 상세 대시보드 ➔ ③ 심층 데이터 리포트로 뎁스(Depth)를 나누어 제공.

# **5\. 데모 시나리오**

| 시연 단계 | 사용자 액션 (데모) | 시스템 반응 (결과) | 심사위원 전달 메시지 (Wow Point) |
| :---- | :---- | :---- | :---- |
| **시연 1 (편의성)** | 양식이 엉망인 A 투자자의 국내 주식 기록(CSV) 업로드 | 3초 만에 핵심 지표가 시각화된 깔끔한 대시보드 생성 | "초보자도 클릭 한 번이면 전문가 수준의 대시보드를 가질 수 있습니다." |
| **시연 2 (일관성)** | 전혀 다른 스키마의 B 투자자 해외 펀드 데이터 추가 업로드 | 기존과 완벽하게 동일한 레이아웃과 분석 기준으로 리포트 생성 | "어떤 데이터가 들어와도 Skills.md가 통제하여 일관된 뷰를 보장합니다." |
| **시연 3 (확장성)** | 코드 수정 없이, Skills.md의 텍스트(예: 위험자산 비중 경고 임계값 50% ➔ 30%)만 수정 | 대시보드 새로고침 시, 차트의 경고등 색상과 문구가 즉시 업데이트됨 | "개발자 없이도 규칙 문서(Text) 수정만으로 서비스 전체가 업데이트되는 혁신입니다." |
