# 금융 투자 대시보드 해커톤 요약

## 🎯 대회 개요

투자 분석 규칙을 정의한 **Skills.md 문서**를 설계하고, 이를 기반으로 **바이브 코딩**을 활용하여 **금융 투자 대시보드 웹 서비스**를 구현하는 해커톤입니다.

- **목표**: 다양한 투자 데이터를 일관된 기준으로 분석·시각화할 수 있는 범용 금융 분석 시스템 설계
- **핵심 산출물**: Skills.md 문서 + 이를 기반으로 자동 생성되는 대시보드 웹 서비스

> ※ Skills는 특정 서비스의 고유 기능이 아니라, **분석 규칙과 생성 지침을 담은 범용 MD 문서**를 의미합니다.

---

## 📄 Skills.md 문서의 역할 (예시)

- 투자 데이터 분석 기준 정의
- 투자 지표 계산 규칙 정의
- 시각화 선택 기준 정의
- 리포트 구성 흐름 설계
- 투자 인사이트 생성 규칙 정의

## 🏆 평가 기준

### 1차 평가 — 대중 투표
최종 제출 팀을 대상으로 투표를 진행해 **상위 10팀**을 선정합니다.

| 구분 | 가중치 |
|---|---|
| 제출팀 투표 | 60% |
| 참가팀 투표 | 20% |
| 대중 투표 | 20% |

**동점자 처리 기준**
1. 참가팀 투표 점수가 높은 팀 우선
2. 산출물 최초 업로드 시간이 빠른 팀 우선

### 2차 평가 — 내부 심사위원 정성 평가
1차 통과한 상위 10팀을 대상으로 진행하며, **최종 순위는 심사위원 평가 점수로 결정**됩니다.

| 평가항목 | 평가 내용 | 배점 |
|---|---|---|
| 범용성 | 투자 데이터 구조 대응 가능성, 재사용 가능성 | 25 |
| Skills.md 설계 | 분석 규칙 명확성, 시각화 기준, 인사이트 생성 규칙 | 25 |
| 대시보드 자동 생성 | 자동 분석 동작 여부, 시각화 적절성, UI 완성도 | 25 |
| 바이브코딩 활용 | 문서 기반 생성 구조, 자동 생성 수준, 수동 구현 최소화 | 15 |
| 실용성 및 창의성 | 실제 활용 가능성, 확장 기능, UX 개선 | 10 |
| **합계** | | **100** |

## 📦 제출물 규칙

### 1. 기획서 (PDF)
구현할 투자 대시보드 서비스의 기획서. 다음 내용 포함:
- 서비스 개요
- 분석 흐름 설계
- 대시보드 구성
- Skills.md 설계 방향
- 확장 기능 아이디어

### 2. Skills.md 문서 (.md → PDF 변환 제출)
투자 분석 규칙을 정의한 문서.
- **1개 또는 여러 개** 제출 가능
- 여러 개일 경우 기능별·역할별로 구분, **.zip으로 제출**
- 포함 예시: 데이터 분석 규칙, 시각화 선택 기준, 대시보드 구성 규칙

### 3. 웹 페이지 (웹 링크)
Skills.md 기반으로 바이브 코딩을 활용해 구현한 대시보드 웹 서비스.
- **배포 URL (필수)**
- GitHub 저장소 링크 (선택)

---

## ⚙️ 개발 / 배포 규칙

- 배포 URL은 외부 접속이 가능해야 하며, **심사 기간 동안 접근 가능**해야 함
- 기술 스택은 자유
- 더미 데이터 또는 공개 데이터 활용 가능
  - 예: 주가, ETF, 포트폴리오, 재무 지표, 시장 지표, 거래 데이터 등
- 외부 API 사용 시 **심사자가 별도 키 없이 확인 가능**해야 함

---

## ⚖️ 공정성 / 저작권

- 제출물(코드, 이미지, 폰트, 아이콘 등)은 저작권·라이선스 준수 필수
- 무단 복제·도용 또는 라이선스 위반 시 **실격 또는 감점**
- 팀 내 역할/분업 자유, **생성형 AI 도구 사용 허용**

---

## ⚠️ 심사 유의사항

- 심사는 기본적으로 **배포 URL 기준**으로 진행
- 확인이 어려운 기능은 심사에서 제외될 수 있음
- 운영진의 추가 공지가 있을 경우 **공지 내용이 우선 적용**
